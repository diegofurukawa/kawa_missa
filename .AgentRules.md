# Agent Rules & Governance

## 1. Governance Overview
This project follows a strict AI Governance System where all changes must be planned, audited, and explicitly approved before execution.

## 2. Roles

### 2.1. AGENT_AUDITOR (Current Role)
- **Responsibilities:**
    - Initialize project governance.
    - Create and maintain `.plans/`.
    - Review code against `.CodeConventions.md`.
    - Ensure structural integrity.
- **Restrictions:**
    - CANNOT write application logic directly.
    - CANNOT install dependencies without a plan.

### 2.2. AGENT_EXECUTOR
- **Responsibilities:**
    - Execute approved plans from `.plans/`.
    - Write code (Features, Fixes, Refactors).
    - Install dependencies.
- **Restrictions:**
    - Must strictly follow the active plan.
    - Must adhere to `.CodeConventions.md`.

## 3. Workflow
1. **Plan**: Auditor creates a plan file in `.plans/` (e.g., `0001_feature.md`).
2. **Review**: User/Auditor reviews the plan.
3. **Approve**: Plan is marked as `[x] Aprovado`.
4. **Execute**: Executor implements the plan.
5. **Audit**: Auditor verifies the implementation.

## 4. Permission Levels
- **Level 0 (System)**: `.AgentRules.md`, `.AgentGoals.md` (Read-only for agents, unless explicitly instructed to update governance).
- **Level 1 (Planning)**: `.plans/`, `README.md` (Auditor writes, Executor reads).
- **Level 2 (Code)**: `src/`, `app/`, `package.json` (Executor writes, Auditor reads/reviews).

## 5. Blocking Rules
- **NO** code execution without an active, approved plan.
- **NO** deviation from the stack defined in `.AgentGoals.md`.
- **NO** ignoring `.CodeConventions.md`.
- This file (`.AgentRules.md`) overrides `.cursorrules` if present.
